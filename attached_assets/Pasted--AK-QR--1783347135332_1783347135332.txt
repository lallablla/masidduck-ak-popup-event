마시떡 AK플라자 수원점 팝업 이벤트용 모바일 원페이지 랜딩페이지를 만들어줘.

중요:
이 페이지는 오프라인 팝업 매장 고객이 QR코드를 스캔해서 들어오는 이벤트 참여 페이지야.
고객은 이 페이지에서 인스타그램, 유튜브, 네이버, 카카오톡 중 하나를 선택해서 이벤트에 참여하고, 완료 화면을 직원에게 보여주면 증정 떡을 받는 구조야.

또한 운영자가 현장에 없을 수도 있어서, 모바일에서도 관리자 비밀번호 4자리만 입력하면 페이지 내용을 바로 수정할 수 있어야 해.
코드 파일을 직접 수정하는 방식이 아니라, 랜딩페이지 안에서 관리자 모드로 들어가서 공지 문구와 이벤트 상태를 수정할 수 있게 만들어줘.

브랜드 정보:
- 브랜드명: 마시떡
- 행사명: 마시떡 AK플라자 수원점 팝업 EVENT
- 행사 기간: 2026.07.10(금) - 2026.07.23(목)
- 장소: AK플라자 수원점
- 이벤트 혜택: SNS 이벤트 참여 시 증정 떡 1개 무료 증정
- 참여 조건: 아래 이벤트 중 하나만 참여해도 OK
- 참여 후 완료 화면을 직원에게 보여주면 증정

디자인 톤:
- 따뜻한 크림 베이지 배경
- 딥레드 / 코랄오렌지 포인트
- 정갈하고 프리미엄한 떡 브랜드 느낌
- 너무 유치한 쿠폰 이벤트 느낌 금지
- 백화점 팝업에 어울리는 깔끔한 모바일 페이지
- 버튼은 크고 누르기 쉽게
- QR로 들어온 고객이 5초 안에 무엇을 해야 하는지 알 수 있게 구성

QR 찍고 원하는 이벤트 하나만 참여하면 OK!

강조 문구:
증정 떡 1개 무료 증정

날짜 배지:
7.10(금) - 7.23(목)

2. 현재 공지 영역
상단 바로 아래에 “오늘의 공지” 박스를 넣어줘.
이 공지는 관리자 모드에서 모바일로 바로 수정 가능해야 해.

기본 공지 문구:
현재 이벤트 진행 중입니다.
아래 중 하나만 참여하신 뒤 완료 화면을 직원에게 보여주세요.
증정 떡은 현장 상황에 따라 변경되거나 조기 소진될 수 있습니다.

관리자 모드에서 수정 가능한 항목:
- 오늘의 공지 제목
- 오늘의 공지 본문
- 이벤트 진행 상태
  예: 진행 중 / 일시 중단 / 증정품 소진 / 빠른 참여만 진행 중

  예: “현재는 카카오톡 채널추가 또는 네이버 저장·알림받기 참여를 추천드립니다.”

STEP 2. 팔로우 / 구독 / 저장 / 채널추가 중 하나 참여
STEP 3. 완료 화면을 직원에게 보여주기

6. 유의사항 영역
작게 넣어줘.

- 이벤트는 1인 1회 참여 가능합니다.
- 참여 완료 화면을 직원에게 보여주셔야 증정이 가능합니다.
- 증정 떡은 현장 상황에 따라 변경되거나 조기 소진될 수 있습니다.
- 이벤트 내용은 현장 상황에 따라 변경될 수 있습니다.

관리자 모드 기능:

페이지 하단에 아주 작게 “관리자” 버튼을 넣어줘.
일반 고객에게는 크게 눈에 띄지 않게 해줘.

관리자 버튼을 누르면 4자리 비밀번호 입력창이 뜨게 해줘.
기본 관리자 비밀번호는 0710으로 설정해줘.

비밀번호가 맞으면 관리자 편집 패널이 열리게 해줘.

관리자 편집 패널에서 수정 가능한 항목:
1. 오늘의 공지 제목
2. 오늘의 공지 본문
3. 이벤트 진행 상태
4. 강조 안내 문구
5. 인스타그램 버튼 노출 여부
6. 네이버 저장·알림받기 버튼 노출 여부
7. 카카오톡 채널추가 버튼 노출 여부
8. 유튜브 구독 버튼 노출 여부

예를 들어 사람이 너무 몰리면 관리자가 인스타그램과 유튜브 버튼을 숨기고,
네이버 저장·알림받기와 카카오톡 채널추가만 보이게 설정할 수 있어야 해.

관리자 편집 패널에는 아래 버튼이 있어야 해.
- 저장하기
- 기본값으로 되돌리기
- 닫기

저장한 내용은 페이지를 새로고침해도 유지되게 해줘.
가능하면 localStorage를 사용해줘.

단, localStorage는 같은 기기에서만 유지되니까,
나중에 다른 기기에서도 동일하게 반영하려면 Firebase나 Supabase 같은 데이터 저장 기능이 필요하다는 안내 주석도 코드에 남겨줘.

가능하다면 Replit 환경에서 간단하게 사용할 수 있도록,
서버 없이 HTML/CSS/JavaScript만으로 먼저 구현해줘.
하지만 “여러 기기에서 관리자 수정 내용이 동시에 반영되는 버전”이 필요하면,
다음 단계에서 Firebase 또는 Supabase 연동 버전으로 확장할 수 있게 구조를 깔끔하게 만들어줘.

추가 요구:
- 모바일 우선 반응형 디자인
- 고객용 화면은 아주 단순하고 직관적으로
- 관리자 화면은 모바일에서도 수정하기 쉽게
- 버튼은 손가락으로 누르기 쉽게 높이 충분히
- 공지 박스는 상단에서 바로 보이게
- 이벤트 상태가 “일시 중단”이나 “증정품 소진”일 때는 버튼보다 공지 문구가 더 먼저 보이게
- 전체 페이지는 빠르게 로딩되게
- 이미지가 없어도 완성도 있게 보이게
- 필요하면 제품 이미지 영역은 선택사항으로 넣되, 없어도 레이아웃이 깨지지 않게 해줘.

최종 결과:
고객이 QR로 접속해서 이벤트에 참여할 수 있는 모바일 랜딩페이지를 만들어줘.
관리자는 페이지 하단의 관리자 버튼을 통해 4자리 비밀번호 0710 입력 후 모바일에서 바로 공지와 버튼 노출 상태를 수정할 수 있어야 해.
